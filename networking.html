1. DNS (Domain Name System)

When you type memefiele1.github.io/cn-project-1 into a browser, DNS translates that domain name into an IP address so your browser knows which server to connect to. Running nslookup memefiele1.github.io returns four IPv4 addresses — GitHub Pages uses multiple servers for load balancing.

2. IP Addressing

This site resolves to both IPv4 and IPv6 addresses. The IPv4 address 185.199.110.153 is a 32-bit address in dotted-decimal format. The IPv6 address 2606:50c0:8001::153 is a 128-bit address supporting modern dual-stack networking.

Record Type Address Description
A 185.199.110.153 IPv4 — 32-bit dotted decimal
AAAA 2606:50c0:8001::153 IPv6 — 128-bit, dual-stack

3. HTTP vs HTTPS

This site is served exclusively over HTTPS on port 443, confirmed by the browser lock icon and Chrome DevTools showing Remote Address: 185.199.110.153:443. The status code returned is 304 Not Modified, meaning the browser cache was validated. TLS encrypts all communication between your browser and GitHub's servers.

Protocol Port Encrypted Status
HTTP 80 ❌ No Redirects to HTTPS
HTTPS 443 ✅ TLS Active — 304 Not Modified

4. Performance Optimizations Applied

This page uses several networking-aware optimizations:

  • Lazy loading — images use loading="lazy" and fade in via IntersectionObserver, reducing initial page load bytes
  • Cache-Control: max-age=86400 — tells browsers to cache assets for 24 hours, cutting repeat-visit load time
  • Minified CSS — styles inlined and compressed to reduce HTTP requests
  • CDN delivery — GitHub Pages serves assets from a global CDN (Fastly) with edge caching
